## 2. 总体方法定位

### 2.1 从集成学习角度解释双 PTM

论文不再将 WavLM 和 UniSpeech-SAT 简单描述为两个独立 PTM，也不再把方法重点放在“为什么必须选择 dual-PTM”上。

修改后的定位是：

- 两个语音预训练模型是一个小型的 **heterogeneous ensemble**；
- 每个 PTM 是一个 ensemble member；
- 两个成员分别生成一个聚类分区；
- 不同预训练目标带来的表示差异构成 ensemble diversity；
- 对齐后的跨模型聚类赋值一致性是可观测的 ensemble consensus signal；
- 模型间不一致表示当前伪标签依赖于具体表示空间，但不直接等价于标签错误。

摘要中增加了如下核心表述：

> From an ensemble-learning perspective, two heterogeneous speech pre-trained models serve as complementary ensemble members and produce two clustering partitions.

引言中将初始伪标签生成重新表述为：

> CGPL formulates initial pseudo-label generation as heterogeneous ensemble inference.

### 2.2 新增集成学习相关工作

在 Related Work 中新增：

```latex
\subsection{Ensemble Learning and Cross-Model Consensus}
```

该小节说明：

- 集成学习同时依赖成员能力和成员差异性；
- 在没有真实标签的无监督学习中，跨成员一致性可以作为决策稳定性的可观测指标；
- 聚类标签具有 permutation invariance，因此必须先完成 cluster-label alignment，才能比较两个分区；
- agreement 不能证明标签必然正确，只能说明该赋值在两个表示空间中具有稳定性。

在 `ref.bib` 中新增两条集成学习经典参考文献：

1. Dietterich, T. G., “Ensemble methods in machine learning,” 2000.
2. Zhou, Z.-H., *Ensemble Methods: Foundations and Algorithms*, 2012.

### 3.2 Agreement-Guided 与 Consensus-Guided 的区分

最终保留 **Consensus-Guided** 作为框架层名称，不建议用 Agreement-Guided 指代整个框架。

统一后的概念层次如下：

| 层次 | 统一术语 | 含义 |
| --- | --- | --- |
| 框架层 | consensus-guided | 利用异构集成成员提供的共同证据指导伪标签构建与训练 |
| 操作层 | cross-model assignment agreement | 对齐两个聚类分区后使用的具体二元判据 |
| 样本层 | cross-view consistent | 两个模型对样本给出相同的对齐后聚类赋值 |
| 样本层 | cross-view discrepant | 两个模型对样本给出不同的对齐后聚类赋值 |
| 数据子集 | consensus-selected subset | 通过跨模型一致性判据选出的伪标签监督子集 |
| 数据子集 | cross-view-disagreement subset | 未获得跨模型一致性支持、只参与无标签目标的子集 |

因此，agreement 是形成 ensemble consensus 的具体判据，而 consensus 是更高层的方法概念。

## 4. 不确定性与错误类型的重新定义

### 4.1 删除含混的 uncertain sample

原稿混用了以下表述：

- uncertain sample；
- low-consensus sample；
- ambiguous sample；
- reliable sample；
- agreement-reliable sample。

这些名称容易产生以下歧义：

- 是语音样本本身不确定，还是它的伪标签不确定？
- disagreement 是否一定意味着伪标签错误？
- agreement 是否足以证明标签正确？

修改后，论文明确指出：

> We use pseudo-label uncertainty to describe uncertainty in the assigned pseudo-label, rather than an intrinsic property of the speech sample.

同时明确：

> Discrepancy indicates a model-dependent pseudo-label assignment, but does not imply that the sample itself is inherently uncertain or that its pseudo-label is necessarily incorrect.

### 4.2 统一两类伪标签问题

问题定义中统一使用以下两个概念：

1. **Assignment noise**：一个样本获得的伪标签可能与其潜在说话人身份不一致。
2. **Speaker-cluster over-segmentation**：同一说话人的语音被划分到多个伪标签类别。

删除或替换了容易与其他聚类概念混淆的 `category fragmentation`。

### 4.3 子集符号修改

保留：

```latex

```

表示 consensus-selected subset。

将原来的：

```latex
U_u
```

修改为：

```latex
U_d
```

表示 cross-view-disagreement subset。这样避免使用 `u` 暗示 uncertain sample。

### Hungarian alignment 边界条件

原稿没有说明两个聚类分区类别数不同时如何使用 Hungarian algorithm，也没有定义未匹配类别的处理方式。

修改后规定：

- 使用零值 dummy entries 补齐 overlap matrix；
- 与 dummy entry 匹配的 cluster 被视为 unmatched；
- unmatched cluster 中的样本不能满足 consensus criterion；
- 未匹配的 aligned label 记为 `\varnothing`。

对应定义为：

```latex
Y_B^{\mathrm{align}}(x_k)
=
\begin{cases}
\mathrm{Map}(Y_B(x_k)), & Y_B(x_k)\text{ is matched},\\
\varnothing, & \text{otherwise}.
\end{cases}
```

## 6. DLNAS 修改

### 6.1 不再使用 correction

方法统一命名为：

> Dynamic Label-Noise-Aware Supervision (DLNAS)

论文明确指出 DLNAS：

- 不替换伪标签；
- 不更新当前训练步骤中的伪标签；
- 不将模型预测假设为始终比聚类伪标签可靠；
- 只在目标函数层面加入 confidence-weighted auxiliary self-training term。

核心表述修改为：

> DLNAS does not modify the selected pseudo-label: it neither replaces nor updates the label during an optimization step.

因此不再将该方法描述为 label correction。

### 6.2 reliability 改为 confidence

模型最大后验概率并不能严格证明预测正确，也不一定是校准后的可靠性概率。因此以下名称进行了修改：

| 原表述 | 修改后表述 |
| --- | --- |
| Global Prediction Reliability | Global Confidence Schedule |
| Sample-Level Posterior-Margin Reliability | Sample-Level Posterior-Margin Confidence |
| global reliability estimate | global confidence schedule |
| reliability score | confidence score |
| reliability-weighted auxiliary target | confidence-weighted auxiliary target |

同时增加限制性说明：

> We use confidence here as an optimization signal and do not interpret it as a calibrated probability that the predicted class is correct.

### 6.3 增加 stop-gradient 说明

新增如下实现约束：

> The predicted label $\hat{y}_i$ and coefficient $\lambda_i$ are detached from the computation graph when the auxiliary term is evaluated.

这样可以避免模型通过直接改变自身权重系数来降低辅助目标，而不是学习更合理的分类表示。

## 13. 当前仍需作者确认的问题

### 13.1 Vox1-O test-set tuning 风险

当前 Ensemble Member Configuration 中写道：

> To instantiate the heterogeneous ensemble, we conduct a layer-wise evaluation on the Vox1-O test set ...

随后根据 Vox1-O 的 EER 选择 WavLM 第 11 层和 UniSpeech-SAT 第 9 层。这可能被 TASLP 审稿人视为：

- 使用测试集进行模型或超参数选择；
- test-set tuning；
- evaluation leakage；
- 最终 EER 可能对 Vox1-O 存在选择偏差。

该问题不能仅通过修改英文措辞解决，需要确认真实实验流程。

建议的合规处理方式包括：

1. 使用 VoxCeleb2 training set 中划分出的 held-out development subset 选择 PTM 和层；
2. 使用独立开发集进行选择；
3. 根据已有文献或预先规定的配置固定层，不使用 Vox1-O 的结果进行选择；
4. 如果 Vox1-O 结果只是事后分析而不是选择依据，应重新撰写该小节并提供真实的选择依据。

## 15. 最终命名建议

当前推荐保留：

```text
Consensus-Guided Pseudo-Labeling (CGPL)
```

原因是该名称可以覆盖完整框架：

- heterogeneous ensemble construction；
- cross-model agreement；
- consensus-based sample selection；
- pseudo-label merging；
- DLNAS；
- disagreement-sample utilization。
